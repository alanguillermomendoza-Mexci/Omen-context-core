



**Omen Context Core**



Omen Context Core is a local-first contextual memory system designed to preserve project continuity during AI-assisted technical workflows.



This project was born from a real operational problem: when working extensively with AI models during project development, context degrades over time. The model begins to forget prior decisions, lose structural continuity, mix different parts of the project or respond outside the established framework. That creates friction, forces reconstruction of previous work and turns continuity into a serious bottleneck.



Omen Context Core was created as a direct response to that limitation. Its purpose is to store important project context in a local, structured and persistent system controlled entirely by the user, instead of relying only on the unstable memory of a conversation.



**Core Principles**



\- Local-first architecture

\- Complete offline operation

\- No cloud dependency

\- No telemetry

\- Structured contextual memory

\- Modular design for future expansion



**Current Purpose**



The current version focuses on one immediate goal: preserving project context in a structured SQLite database so that work can be resumed when AI loses track of previous decisions or project state.



**Repository Scope**



This repository contains the public architecture of the project:

\- SQL schema

\- database creation scripts

\- public documentation

\- future support scripts



It does not contain the real working database or private contextual entries. Those remain local and are intentionally excluded from version control.
